Important Concepts

Workflow .yml

The complete automated process defined in a YAML file. It contains the configuration for all jobs and steps.

Runner

The server that executes your jobs. Can be GitHub-hosted (e.g., ubuntu-latest) or self-hosted on your own infrastructure.

Job

A group of steps that execute on the same runner. Jobs run in parallel by default but can be configured to run sequentially.

Trigger on:

The event that starts the workflow. Common triggers include push, pull_request, or scheduled times.

Step

A single task within a job. It can either run a shell command or execute a pre-built GitHub Action.

Secrets / Env

Encrypted environment variables used to store sensitive data like API keys, tokens, and passwords safely.
