GitHub Actions Folder Structure

my-project/
├── .github/
│ └── workflows/
│ └── ci.yml
├── src/
│ └── app.js
├── package.json
└── README.md

Standard Location

GitHub specifically looks for workflow configurations in the .github/workflows/ directory.

Configuration File

The ci.yml (or any .yml file) contains the automation instructions. You can have multiple files here for different workflows (e.g., release.yml, test.yml).

Why this structure?

Keeping workflows in a hidden dot-folder prevents clutter in your main source code while ensuring GitHub's runner knows exactly where to find instructions.
